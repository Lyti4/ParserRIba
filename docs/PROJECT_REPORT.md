# ParserRIba: отчет по состоянию проекта

Дата: 2026-05-12

## Краткий вывод

Проект уже можно запускать на Windows в разработческом режиме и собирать в
portable-папку через PyInstaller. Главная рабочая линия сейчас: стабилизировать
Пятерочку на реальном российском IP/прокси, затем довести еще 1-2 магазина до
такого же уровня и только после этого делать публичный релиз.

## Что уже приведено в порядок

- Python 3.11 проверен на Windows.
- Зависимости устанавливаются в локальное окружение.
- Camoufox запускается через общий launcher.
- Для Пятерочки есть отдельный Camoufox smoke-тест.
- Прокси можно задавать через `.env` переменную `PARSER_PROXY`.
- Прокси маскируется в логах, чтобы не светить пароль.
- GeoIP поддерживается через `GeoLite2-City.mmdb`.
- `main.py --list-stores` работает из исходников и из `ParserRIba.exe`.
- `main.py --check-env` добавлен для самодиагностики.
- Portable-сборка создает `dist\ParserRIba\ParserRIba.exe`.
- Сборка создает архив `dist\ParserRIba-windows-x64.zip`.
- В архив добавляется `README_START_HERE.txt`.
- Тесты проходят: `47 passed`.

## Главные риски

1. Антибот Пятерочки может отдавать challenge-страницу вместо каталога.
2. Российский IP/прокси нужен для стабильной проверки сайта.
3. Бесплатные/VPN IP часто дают плохой результат: блокировки, капчи,
   нестабильные сессии.
4. Camoufox не входит внутрь `ParserRIba.exe` как сам браузер. Сейчас программа
   ищет установленный Camoufox по пути `C:\CamoufoxBrowser\...`.
5. `GeoLite2-City.mmdb` большой файл. Его удобно класть рядом с exe, но перед
   публичной публикацией нужно решить вопрос лицензии и размера архива.

## Рекомендованный план до публичного релиза

### Этап 1. Пятерочка

- Проверить smoke-тест с российским резидентским или мобильным прокси.
- Зафиксировать в отчете:
  - HTTP статус;
  - финальный URL;
  - заголовок страницы;
  - найден ли антибот;
  - сколько товаров извлечено.
- Если challenge сохраняется, добавить ротацию прокси-сессии после 403/капчи.
- Проверить `--store pyaterochka --no-headless` в portable exe.

### Этап 2. Пользовательский запуск

- Сделать простой GUI launcher:
  - поле прокси;
  - выбор магазина;
  - кнопка проверки окружения;
  - кнопка запуска;
  - просмотр последних логов;
  - папка результатов.
- Сохранять настройки в `.env` или `config.local.yaml`, не в Git.

### Этап 3. Еще 1-2 магазина

- Выбрать магазины с меньшей антибот-защитой.
- Довести их до одинакового контракта:
  - `async def parse(...) -> list[Product]`;
  - ошибки категории логируются и не роняют весь запуск;
  - URL и селекторы только из `knowledge_base/`.

### Этап 4. Релиз

- Проверить ZIP на чистой Windows-машине.
- Добавить GitHub Actions для тестов.
- Создать GitHub Release и приложить `ParserRIba-windows-x64.zip`.
- После первых пользователей собрать ошибки из `--check-env` и логов.

### Этап 5. Установщик

- После стабильного ZIP сделать Inno Setup или NSIS installer.
- Установщик должен:
  - создавать папку программы;
  - класть ярлык на рабочий стол;
  - открывать README после установки;
  - не сохранять прокси-пароли в публичных файлах.

## Команды проверки

Из исходников:

```powershell
.\.venv\Scripts\python.exe main.py --check-env
.\.venv\Scripts\python.exe main.py --list-stores
.\.venv\Scripts\python.exe -m pytest -q
```

После сборки:

```powershell
dist\ParserRIba\ParserRIba.exe --check-env
dist\ParserRIba\ParserRIba.exe --list-stores
```

Сборка:

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\scripts\build_windows.ps1 -Clean
```

## Что не публиковать

- `.env`;
- реальные прокси-строки;
- логи с чувствительными данными;
- временные HTML/PNG smoke-артефакты;
- приватные ключи MaxMind;
- локальные виртуальные окружения.

## Следующее лучшее действие

Запустить Пятерочку через portable exe с рабочим российским прокси и сохранить
smoke-отчет. Если будет антибот, первым делом дорабатывать не интерфейс, а
ротацию сессий/прокси и восстановление после challenge.

## Daijro/Camoufox backlog

Реализуем только то, что помогает ParserRIba, без платных браузеров, LLM,
облаков и внешних API.

### Уже внедрено

- `camoufox[geoip]` как основной браузерный антидетект-слой.
- `browserforge` как зависимость для отпечатков.
- Единый Camoufox launcher.
- Локальный `CAMOUFOX_BIN`.
- `geoip=True` при наличии `GeoLite2-City.mmdb`.
- `humanize=True`.
- `block_images=True`.
- `i_know_what_im_doing=True`.
- BrowserForge fingerprint profile ограничен по Windows через единый Camoufox launcher.
- Прокси через `PARSER_PROXY`.
- Маскировка прокси в логах.
- Smoke-отчет с внешним IP браузера.
- Human-like поведение вынесено в `utils/human_behavior.py`.
- Для Пятерочки подключены случайные паузы, глубина скролла, micro-scroll и hover по карточкам.
- Smoke-отчет показывает fingerprint и behavior profile.

### Нужно сделать дальше

### Пока не берем

- Прямое копирование `geoip-all-in-one`: у него GPL-3.0, поэтому используем
  локальный `.mmdb`, а не переносим код.
- Платные browser API и облачные антикапчи.

## Captcha handling

Автоматическое разгадывание капчи не внедряем. Для Пятерочки поддерживаем
ручной режим: видимый Camoufox, загрузка изображений и пауза после открытия
страницы. Это позволяет проверить, что прокси/GeoIP/браузерный профиль работают,
без интеграции антикапча-сервисов.

Команда:

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\scripts\run_pyaterochka_visual.ps1
```
