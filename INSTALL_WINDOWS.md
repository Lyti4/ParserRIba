# 🪟 Установка и запуск парсера на Windows

## Шаг 1: Установка Python

1. Откройте https://www.python.org/downloads/
2. Скачайте **Python 3.9 или выше** (рекомендуется последняя версия)
3. Запустите установщик
4. ✅ **ВАЖНО:** Поставьте галочку **"Add Python to PATH"** перед установкой!
5. Нажмите "Install Now"

### Проверка установки

Откройте **Командную строку** (cmd) или **PowerShell** и выполните:

```bash
python --version
```

Должно отобразиться что-то вроде `Python 3.11.5`

---

## Шаг 2: Скачивание проекта

### Вариант A: Через Git (рекомендуется)

1. Установите Git: https://git-scm.com/download/win
2. Откройте PowerShell в папке, куда хотите скачать проект
3. Выполните:
   ```bash
   git clone <URL_вашего_репозитория>
   cd <имя_репозитория>
   ```

### Вариант B: Ручное копирование

1. Скопируйте все файлы из репозитория в папку, например:
   ```
   C:\Projects\fish-parser\
   ```

---

## Шаг 3: Создание виртуального окружения

Откройте PowerShell или cmd в папке проекта:

```bash
cd C:\Projects\fish-parser
```

Создайте виртуальное окружение:

```bash
python -m venv venv
```

Активируйте его:

```bash
venv\Scripts\activate
```

В начале строки появится `(venv)` — это значит, что окружение активировано.

---

## Шаг 4: Установка зависимостей

Убедитесь, что виртуальное окружение активно (видите `(venv)` в начале строки).

Установите все необходимые библиотеки:

```bash
pip install -r requirements.txt
```

Это займёт 1-3 минуты. Дождитесь завершения установки.

---

## Шаг 5: Настройка (опционально)

Если хотите изменить настройки:

1. Скопируйте файл `.env.example` в `.env`:
   ```bash
   copy .env.example .env
   ```

2. Откройте `.env` в блокноте и измените параметры при необходимости:
   - `STORES` — какие магазины парсить
   - `REQUEST_DELAY` — задержка между запросами (увеличьте если есть ошибки)

Для начала можно пропустить этот шаг — будут использоваться настройки по умолчанию.

---

## Шаг 6: Запуск парсера

Убедитесь, что:
- Вы подключены к интернету
- Вы запускаете с **домашнего/офисного компьютера** (не сервер!)
- Виртуальное окружение активировано (видите `(venv)`)

Запустите парсер:

```bash
python main.py
```

### Что произойдёт:

1. Парсер начнёт собирать данные из магазинов
2. В консоли будет отображаться прогресс
3. Результаты сохранятся в файл `fish_products.xlsx`

Процесс займёт **5-15 минут** в зависимости от количества магазинов.

---

## 📊 Просмотр результатов

После завершения откройте файл `fish_products.xlsx` в Excel.

Вы увидите таблицу с колонками:
- **name** — название товара
- **price** — цена в рублях
- **store** — магазин
- **url** — ссылка на товар
- **category** — категория
- **scraped_at** — время сбора данных

## 🛠️ Решение проблем

### Ошибка: `'python' is not recognized`

Python не добавлен в PATH. Переустановите Python с галочкой "Add to PATH"
или используйте полный путь:
```bash
C:\Users\<ВашеИмя>\AppData\Local\Programs\Python\Python311\python.exe -m venv venv
```

### Ошибка: `ModuleNotFoundError: No module named 'curl_cffi'`

Переустановите зависимости:
```bash
pip install --upgrade -r requirements.txt
```

### Ошибка: SSL Handshake Failure (Окей)

Проблемы с SSL. Обновите сертификаты:
```bash
pip install --upgrade certifi
```

Или исключите Окей из парсинга в файле `.env`:
```
STORES=pyaterochka,magnit,perekrestok
```

### Ошибка: 0 товаров найдено

1. **Проверьте IP** — нельзя запускать с сервера/VPS, только домашний компьютер
2. Увеличьте задержку в `.env`: `REQUEST_DELAY=5`
3. Попробуйте запустить только один магазин:
   ```
   STORES=pyaterochka
   ```

### Ошибка: 401 Unauthorized (Лента, Ашан)

Эти сайты требуют авторизации. Исключите их из парсинга:
```
STORES=pyaterochka,magnit,perekrestok
```

## 💡 Советы

1. **Запускайте вечером** — меньше нагрузка на сайты
2. **Начните с 1-2 магазинов** — проверьте работу
3. **Увеличьте REQUEST_DELAY** если много ошибок
4. **Сохраняйте результаты** — файл создаётся после каждого запуска

---

## 📞 Если что-то не работает

1. Посмотрите лог ошибок в консоли
2. Проверьте, что запускаете с домашнего компьютера
3. Попробуйте исключить проблемные магазины
4. Увеличьте задержки между запросами
