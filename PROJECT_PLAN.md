# 📘 План развития проекта ParserRIba

## 🎯 Цель проекта
Создание надежного, масштабируемого и самовосстанавливающегося парсера цен на рыбные товары для российских сетей магазинов (Пятерочка, Магнит, Перекресток, Лента, Ашан, О'Кей), способного обходить современные защиты (Cloudflare, капчи) и адаптироваться к изменениям структуры сайтов без переписывания кода.

---

## 🏗 Архитектурная философия (на основе лучших практик browser-act)

Мы отказываемся от жесткой привязки логики парсинга к коду. Вместо этого мы внедряем **Data-Driven подход**:
1.  **Знания (Knowledge)** отделены от **Логики (Code)**.
2.  **Гибридный движок загрузки**: Быстрый HTTP (curl-cffi) → Fallback на умный браузер (при необходимости).
3.  **Самодиагностика**: Система понимает, когда она заблокирована, и меняет стратегию.

## 🚀 Ключевые компоненты для реализации

### 1. Knowledge Base (`knowledge_base/`)
Вместо хардкода URL и селекторов в коде, мы храним их в Markdown-файлах.
**Структура файла (пример `pyaterochka.md`):**
- **Base URLs**: Основные ссылки и регионы.
- **Selectors**: CSS/XPath селекторы для цены, названия, веса (с запасными вариантами).
- **API Endpoints**: Если сайт использует скрытые API (JSON), здесь хранятся параметры запросов.
- **Anti-Bot Triggers**: Признаки блокировки (редиректы, специфичные слова в HTML, статус 403).
- **Notes**: Особенности (например, "требуется скролл", "цена зависит от карты лояльности").

*Преимущество:* Если сайт изменил верстку, мы правим только `.md` файл, не трогая код парсера.

### 2. Session Manager (`utils/session_manager.py`)
Централизованное управление состоянием.
- **Proxy Rotation**: Поддержка списка прокси, автоматическая смена при ошибках.
- **Cookie Jar**: Сохранение и загрузка cookies между сессиями (эмуляция постоянного пользователя).
- **Fingerprinting**: Генерация реалистичных заголовков (User-Agent, Sec-Ch-Ua) под каждый запрос.
- **Region Management**: Переключение регионов (Москва, СПБ и т.д.) через заголовки или куки.

### 3. Hybrid Site Loader (`utils/site_loader.py`)
Умный механизм загрузки контента:
1.  **Попытка 1 (Fast)**: Запрос через `curl-cffi` с имперсонацией браузера. Быстро, дешево.
2.  **Анализ**: Проверка ответа на наличие триггеров блокировки (из Knowledge Base).
3.  **Попытка 2 (Stealth)**: Если обнаружена блокировка — автоматический подъем headless-браузера (Playwright) со скриптами маскировки (`stealth.js`).
4.  **Попытка 3 (Manual)**: Если автомат не справился — логирование ошибки с просьбой запустить в визуальном режиме.

### 4. Strategies (`strategies/`)
Набор готовых алгоритмов для сложных случаев:
- **ScrollStrategy**: Автоматический скролл страницы до конца для подгрузки товаров (lazy loading).
- **PaginationStrategy**: Умение переходить по страницам каталога (клик или прямая ссылка).
- **WaitStrategy**: Умное ожидание элементов (не просто `sleep`, а ожидание появления конкретного селектора).

### 5. Pydantic Models (`models/product.py`)
Строгая типизация данных товара для гарантии качества экспорта.
```python
class FishProduct(BaseModel):
    store: str
    name: str
    price: int
    old_price: Optional[int]
    weight: str
    unit_price: Optional[float] # Цена за кг
    url: str
    image_url: Optional[str]
    timestamp: datetime
```

## 💡 Принципы разработки (Guidelines)

1.  **Никакого хардкода**: Селекторы и URL только в Knowledge Base.
2.  **Graceful Degradation**: Если не получилось быстро — пробуем медленно и надежно. Если не получилось совсем — сохраняем ошибку и идем дальше, не ломая весь процесс.
3.  **Логирование всего**: Каждое действие, каждая смена прокси, каждая ошибка должны быть в логе.
4.  **Асинхронность**: Все сетевые операции строго асинхронные (`async/await`).
5.  **Чистота кода**: Следование PEP8, использование type hints.

## 🛠 Технический стек
- **Язык**: Python 3.10+
- **HTTP**: `curl-cffi` (для имперсонации), `httpx`
- **Browser**: `playwright` + `playwright-stealth`
- **Parsing**: `beautifulsoup4`, `lxml`, `parsel`
- **Data**: `pydantic`, `pandas`, `openpyxl`
- **Config**: `pydantic-settings`, `dotenv`
- **Logging**: Стандартный `logging` с кастомными форматерами
