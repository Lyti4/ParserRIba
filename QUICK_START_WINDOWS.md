# 🚀 Быстрый старт для Windows - ParserRIba

## Минимум действий: скачал → запустил → получил результат

Эта инструкция позволит вам запустить парсер **без ручной настройки**. Все необходимые настройки уже встроены в программу.

---

## ⚡ ЧТО НУЖНО СДЕЛАТЬ (5 шагов)

### ШАГ 1: Установите Python (если ещё не установлен)

1. Откройте браузер и перейдите на: **https://www.python.org/downloads/**
2. Скачайте последнюю версию Python для Windows
3. **ВАЖНО:** При установке поставьте галочку ✅ **"Add Python to PATH"**
4. Нажмите "Install Now"
5. Дождитесь окончания установки

**Проверка установки:**
```bash
python --version
```
Должна отобразиться версия Python (например, `Python 3.12.0`)

---

### ШАГ 2: Откройте командную строку

Нажмите **Win + R**, введите `cmd` и нажмите **Enter**

Или: нажмите **Пуск** → введите `cmd` → нажмите **Enter**

---

### ШАГ 3: Скачайте проект с GitHub

В командной строке выполните команду:

```bash
git clone https://github.com/Lyti4/ParserRIba.git
```

Дождитесь окончания загрузки. Проект скачается в папку `ParserRIba`.

---

### ШАГ 4: Перейдите в папку проекта и установите зависимости

Выполните **по очереди** следующие команды (копируйте и вставляйте по одной):

```bash
cd ParserRIba
```

```bash
python -m venv venv
```

```bash
venv\Scripts\activate
```

```bash
pip install --upgrade pip
```

```bash
pip install -r requirements.txt
```

```bash
playwright install chromium
```

**Что делают эти команды:**
- `cd ParserRIba` — переход в папку проекта
- `python -m venv venv` — создание виртуального окружения
- `venv\Scripts\activate` — активация виртуального окружения (появится `(venv)` в начале строки)
- `pip install --upgrade pip` — обновление установщика пакетов
- `pip install -r requirements.txt` — установка всех необходимых библиотек
- `playwright install chromium` — установка браузера для парсинга

**Время выполнения:** 2-5 минут (зависит от скорости интернета)

---

### ШАГ 5: Запустите парсер

В той же командной строке (убедитесь, что видите `(venv)` в начале строки) выполните:

```bash
python main.py
```

**ВСЁ!** Парсер начал работу.

## 📁 СТРУКТУРА ПАПОК

После клонирования репозитория у вас будет такая структура:

```
ParserRIba/
├── main.py              ← главный файл (запуск через python main.py)
├── config.py            ← настройки (уже настроены по умолчанию)
├── requirements.txt     ← список библиотек
├── .env.example         ← пример настроек (можно игнорировать)
├── README.md            ← документация
├── models/              ← модели данных
├── parsers/             ← парсеры магазинов
└── utils/               ← вспомогательные функции
```

**Вам НЕ НУЖНО** ничего создавать или настраивать вручную!

## ⚙️ НАСТРОЙКА (ПО ЖЕЛАНИЮ)

По умолчанию парсер работает с оптимальными настройками. Если хотите изменить параметры:

### Изменить список магазинов или имя файла вывода:

1. Создайте файл `.env` в папке `ParserRIba` (необязательно)
2. Добавьте нужные параметры:

```
OUTPUT_FILE=my_fish_products.xlsx
REQUEST_DELAY=5
STORES=pyaterochka,magnit
```

**Но это НЕ ОБЯЗАТЕЛЬНО!** Программа отлично работает с настройками по умолчанию.

## ❗ ВОЗМОЖНЫЕ ПРОБЛЕМЫ И РЕШЕНИЯ

### Проблема: Команда `git` не найдена

**Решение:** Установите Git:
1. Скачайте с https://git-scm.com/download/win
2. Установите с настройками по умолчанию
3. Перезапустите командную строку
4. Повторите шаг 3

---

### Проблема: Ошибка при установке библиотек

**Решение:**
```bash
python -m pip install --upgrade pip
pip install -r requirements.txt --force-reinstall
```

---

### Проблема: Ошибка при запуске `playwright install`

**Решение:**
```bash
pip install playwright
playwright install chromium
playwright install-deps chromium
```

---

### Проблема: Парсер не запускается

**Решение:** Убедитесь, что виртуальное окружение активировано (в начале строки должно быть `(venv)`).

Если нет — выполните:
```bash
cd ParserRIba
venv\Scripts\activate
python main.py
```

---

### Проблема: Товары не находятся

**Возможные причины:**
- Временные проблемы с сайтами магазинов
- Блокировка по IP

**Решение:**
- Подождите 10-15 минут и попробуйте снова
- Проверьте подключение к интернету
- Посмотрите логи в файле `parser.log` (создаётся в папке проекта)
