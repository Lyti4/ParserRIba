# 📘 ПОДРОБНАЯ ИНСТРУКЦИЯ ПО ЗАПУСКУ ПАРСЕРА НА WINDOWS
## Для начинающих (полный гайд "для чайников")

---

## 📋 ЧТО ВАМ ПОНАДОБИТСЯ

Перед началом убедитесь, что у вас есть:
1. **Компьютер с Windows** (10 или 11)
2. **Интернет** для скачивания программ
3. **Несколько ГБ свободного места** на диске
4. **Время**: около 15-30 минут на первоначальную настройку

---

## 🔧 ШАГ 1: УСТАНОВКА PYTHON

### Вариант А: Если Python НЕ установлен

1. **Откройте браузер** (Chrome, Firefox, Edge — любой)

2. **Перейдите на официальный сайт Python:**
   ```
   https://www.python.org/downloads/
   ```

3. **Скачайте установщик:**
   - Сайт автоматически предложит версию для Windows
   - Нажмите большую красную кнопку **"Download Python 3.x.x"** (где x.x — актуальная версия)

4. **Запустите скачанный файл** (например, `python-3.12.0-amd64.exe`)

5. **⚠️ ОЧЕНЬ ВАЖНО!** 
   В первом же окне установки **ПОСТАВЬТЕ ГАЛОЧКУ**:
   ```
   ☑ Add Python to PATH
   ```
   *(Без этой галочки ничего не заработает!)*

6. Нажмите **"Install Now"**

7. Дождитесь окончания установки и нажмите **"Close"**

8. **Проверьте установку:**
   - Нажмите `Win + R` на клавиатуре
   - Введите `cmd` и нажмите Enter
   - В черном окне введите:
     ```
     python --version
     ```
   - Должно появиться что-то вроде: `Python 3.12.0`
   - Если видите версию — всё отлично! Переходите к Шагу 2.
   - Если ошибка — перезагрузите компьютер и попробуйте снова.

### Вариант Б: Если Python уже установлен

Пропустите этот шаг и переходите к Шагу 2.

---

## 📁 ШАГ 2: ПОДГОТОВКА ПАПКИ С ПРОЕКТОМ

1. **Создайте папку для проекта:**
   - Откройте «Мой компьютер» или «Проводник»
   - Перейдите на диск `C:` или в любую удобную папку
   - Создайте новую папку, например:
     ```
     C:\fish-parser
     ```

2. **Скопируйте файлы проекта в эту папку:**
   
   Убедитесь, что в папке `C:\fish-parser` находятся следующие файлы:
   ```
   C:\fish-parser\
   ├── main.py              ← главный файл запуска
   ├── config.py            ← настройки
   ├── requirements.txt     ← список библиотек
   ├── .env.example         ← пример настроек
   ├── .gitignore
   ├── README.md
   ├── INSTALL_WINDOWS.md
   ├── models\              ← папка с моделями данных
   ├── parsers\             ← папка с парсерами магазинов
   └── utils\               ← папка с утилитами
   ```

3. **Создайте файл настроек `.env`:**
   - В папке `C:\fish-parser` найдите файл `.env.example`
   - Скопируйте его и переименуйте копию в `.env`
   - **Важно:** Файл должен называться именно `.env` (без расширения .txt)
   
   *Если не видите расширение файла:*
   - В проводнике нажмите «Вид» → поставьте галочку «Расширения имен файлов»
   - Удалите `.example` из названия, оставив только `.env`

4. **Настройте `.env` (по желанию):**
   - Откройте файл `.env` блокнотом
   - Можете изменить настройки или оставить как есть:
     ```
     OUTPUT_FILE=fish_products.xlsx
     REQUEST_DELAY=3
     REQUEST_TIMEOUT=30
     STORES=pyaterochka,magnit,perekrestok,lenta,auchan,okey
     ```
   - Сохраните и закройте файл

## 📦 ШАГ 4: СОЗДАНИЕ ВИРТУАЛЬНОГО ОКРУЖЕНИЯ

**Зачем это нужно?** Виртуальное окружение изолирует библиотеки проекта от других программ.

1. **Создайте виртуальное окружение:**
   
   В командной строке (находясь в `C:\fish-parser`) введите:
   ```
   python -m venv venv
   ```
   
   Нажмите Enter и подождите 5-10 секунд.
   
   В папке проекта появится новая папка `venv`.

2. **Активируйте виртуальное окружение:**
   
   Введите команду:
   ```
   venv\Scripts\activate
   ```
   
   Нажмите Enter.
   
   **Как понять, что всё получилось?**
   Слева в командной строке появится `(venv)`:
   ```
   (venv) C:\fish-parser>
   ```

   ⚠️ **Важно:** Теперь каждый раз при запуске проекта вам нужно активировать окружение этой командой!

   - Попробуйте выполнить команду повторно

## 🌐 ШАГ 6: УСТАНОВКА БРАУЗЕРА И ДРАЙВЕРОВ PLAYWRIGHT

**Playwright** — это инструмент, который управляет браузером для парсинга сайтов.

1. **Установите браузеры для Playwright:**
   
   В командной строке (с активным `(venv)`) введите:
   ```
   playwright install
   ```
   
   Нажмите Enter.

2. **Дождитесь загрузки браузеров:**
   
   Playwright скачает и установит:
   - Chromium (версия Chrome для автоматизации)
   - Firefox
   - WebKit
   
   Это может занять **5-15 минут** и несколько сотен мегабайт трафика.
   
   Вы увидите прогресс-бары загрузки.

3. **Установите системные зависимости (если потребуется):**
   
   На Windows обычно не требуется, но если будет ошибка — введите:
   ```
   playwright install-deps
   ```

4. **Проверьте установку:**
   
   Введите:
   ```
   playwright --version
   ```
   
   Должна отобразиться версия Playwright.

## ⚙️ ШАГ 9: НАСТРОЙКА ПОД СЕБЯ

### Изменение списка магазинов

Откройте файл `.env` в блокноте и измените строку `STORES`:

```
# Только Пятерочка и Магнит (быстро):
STORES=pyaterochka,magnit

# Все магазины кроме Окей:
STORES=pyaterochka,magnit,perekrestok,lenta,auchan

# Один магазин для теста:
STORES=pyaterochka
```

**Названия магазинов:**
- `pyaterochka` — Пятерочка
- `magnit` — Магнит
- `perekrestok` — Перекресток
- `lenta` — Лента
- `auchan` — Ашан
- `okey` — Окей

### Изменение скорости работы

В файле `.env` измените `REQUEST_DELAY`:

```
# Медленнее, но надежнее (5 секунд между запросами):
REQUEST_DELAY=5

# Быстрее (2 секунды):
REQUEST_DELAY=2
```

**Рекомендация:** Не ставьте меньше 2 секунд — сайты могут заблокировать.

### Режим видимости браузера

В файле `main.py` найдите строку:

```python
VISUAL_MODE = True
```

- `True` — вы видите окно браузера (медленнее, но можно наблюдать)
- `False` — браузер работает в фоне (быстрее, но не видно процесса)

## ❗ ВОЗМОЖНЫЕ ПРОБЛЕМЫ И РЕШЕНИЯ

### Проблема 1: "python не является внутренней или внешней командой"

**Решение:**
- Вы не поставили галочку "Add Python to PATH" при установке
- Переустановите Python, поставив галочку
- ИЛИ добавьте Python в PATH вручную через «Свойства системы» → «Переменные среды»

### Проблема 2: Ошибка при установке библиотек

**Решение:**
- Проверьте интернет-соединение
- Обновите pip:
  ```
  python -m pip install --upgrade pip
  ```
- Попробуйте установить библиотеки по одной

### Проблема 3: Playwright не устанавливается

**Решение:**
- Удалите папку `venv` и создайте заново
- Убедитесь, что у вас достаточно места на диске
- Попробуйте установить браузеры по отдельности:
  ```
  playwright install chromium
  ```

### Проблема 4: Парсер зависает на каком-то магазине

**Решение:**
- Закройте браузер и командную строку (Ctrl+C)
- Увеличьте `REQUEST_TIMEOUT` в `.env` до 60
- Уменьшите количество магазинов в `STORES`
- Проверьте логи в файле `parser.log`

### Проблема 5: Мало товаров или пустой файл

**Решение:**
- Некоторые магазины могут блокировать парсинг
- Попробуйте запустить позже
- Используйте только 1-2 магазина для теста
- Проверьте, работает ли сайт магазина в обычном браузере

### Проблема 6: Антивирус блокирует

**Решение:**
- Добавьте папку `C:\fish-parser` в исключения антивируса
- Временно отключите антивирус на время парсинга
